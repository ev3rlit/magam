## User Scenarios & Testing *(mandatory)*

### User Story 1 - Move Absolute Nodes Safely (Priority: P1)

캔버스 편집자는 절대 좌표 기반 노드를 이동할 때, 코드에서 해당 노드의 좌표 값만 바뀌길 원한다.

**Why this priority**: 가장 빈번한 편집 동작이며, 예기치 않은 코드 변경 위험을 가장 빠르게 줄일 수 있다.

**Independent Test**: 절대 좌표 노드를 드래그해 이동한 뒤 코드 변경 내역을 확인했을 때, 해당 노드의 `x`, `y` 값만 변경되면 독립적으로 검증 가능하다.

**Acceptance Scenarios**:

1. **Given** 절대 좌표 노드가 캔버스에 존재하고, **When** 사용자가 노드를 다른 위치로 이동하면, **Then** 시스템은 동일 노드의 좌표 값만 갱신한다.
2. **Given** 노드 이동이 완료되었고 소스가 최신 상태이며, **When** 변경 내역을 확인하면, **Then** 좌표 값 외의 속성 및 구조 변경이 없어야 한다.
3. **Given** 좌표를 직접 갱신할 수 없는 노드 표현이 감지되면, **When** 사용자가 이동을 시도하면, **Then** 시스템은 부분 반영 없이 편집 거부 사유를 명확히 안내한다.
4. **Given** 편집 중 소스가 외부 변경되어 기준 버전이 달라지면, **When** 사용자가 반영을 시도하면, **Then** 시스템은 저장을 거부하고 재동기화(재로드) 안내를 제공해야 한다.
5. **Given** 사용자가 노드 이동 편집을 완료하면, **When** undo를 1회 실행하면, **Then** 해당 편집 완료 이벤트 1건만 되돌려져야 한다.
6. **Given** 사용자가 undo를 1회 실행한 상태에서, **When** redo를 1회 실행하면, **Then** 직전 undo된 편집 완료 이벤트 1건만 복원되어야 한다.
7. **Given** 렌더 결과 전체에서 식별자 전역 유일성이 깨진 상태라면, **When** 편집 반영을 시도하면, **Then** 시스템은 반영을 거부하고 중복 해결 안내를 제공해야 한다.

### User Story 2 - Edit Textual Children in Place (Priority: P2)

캔버스 편집자는 Markdown 또는 일반 텍스트를 children으로 가진 오브젝트의 텍스트를 캔버스에서 직관적으로 수정하고, 코드에서는 텍스트 부분만 바뀌길 원한다.

**Why this priority**: 콘텐츠 수정은 좌표 이동 다음으로 빈도가 높고, 문서 품질과 협업 효율에 직접 영향을 준다.

**Independent Test**: Markdown 편집 시 WYSIWYG로 결과를 확인하며 텍스트를 수정한 뒤 코드 변경 내역을 확인했을 때, 선택한 오브젝트의 children 내용만 변경되고 다른 오브젝트/속성이 유지되면 독립적으로 검증 가능하다.

**Acceptance Scenarios**:

1. **Given** Markdown 또는 텍스트 children을 가진 오브젝트가 있고, **When** 사용자가 텍스트를 수정하면, **Then** 시스템은 해당 children 텍스트만 갱신한다.
2. **Given** 텍스트가 여러 줄 또는 마크다운 서식을 포함하고 있고, **When** 사용자가 일부 구간을 편집하면, **Then** 시스템은 사용자가 수정한 내용만 반영하고 관련 없는 내용은 유지한다.
3. **Given** 사용자가 Markdown 콘텐츠를 편집 중이고, **When** 서식 요소를 입력 또는 수정하면, **Then** 시스템은 WYSIWYG 방식으로 시각 결과를 즉시 확인할 수 있게 해야 한다.
4. **Given** 캔버스에 텍스트/마크다운 오브젝트가 여러 개 있고, **When** 사용자가 특정 오브젝트 하나를 선택해 편집하면, **Then** 선택된 오브젝트만 편집되고 나머지 오브젝트는 변경되지 않아야 한다.
5. **Given** 동일 오브젝트에 텍스트 외 속성이 존재하고, **When** 텍스트 편집이 완료되면, **Then** 다른 속성과 오브젝트 구조는 변경되지 않는다.
6. **Given** 사용자가 텍스트 편집을 완료하면, **When** undo를 1회 실행하면, **Then** 해당 텍스트 편집 완료 이벤트 1건만 되돌려져야 한다.

### User Story 3 - Reposition Attached Sticker and Washi Tape Relatively (Priority: P3)

캔버스 편집자는 부모 오브젝트에 attach된 스티커/와시테이프를 이동할 때 절대 좌표 재작성 없이 상대 위치만 조정하길 원한다.

**Why this priority**: 장식 요소의 미세 조정 품질을 높이면서도 기존 attach 모델을 유지해 재편집 가능성을 보장한다.

**Independent Test**: attach된 스티커/와시테이프를 이동한 뒤 코드 변경 내역을 확인했을 때, 상대 오프셋 값만 변경되고 부모-자식 attach 관계가 유지되면 독립적으로 검증 가능하다.

**Acceptance Scenarios**:

1. **Given** 스티커 또는 와시테이프가 부모에 attach되어 있고, **When** 사용자가 위치를 조정하면, **Then** 시스템은 상대 위치 값만 갱신한다.
2. **Given** 상대 위치 조정이 완료되었고, **When** 코드를 재로딩해 캔버스를 다시 열면, **Then** attach 관계는 유지된 채 조정된 위치가 재현되어야 한다.
3. **Given** attach 대상 부모가 유효하지 않은 상태라면, **When** 상대 위치 반영을 시도하면, **Then** 시스템은 반영을 중단하고 유효한 대상 확인을 안내한다.
4. **Given** attach 상대 위치 편집이 완료되면, **When** undo를 1회 실행하면, **Then** 해당 상대 위치 편집 완료 이벤트 1건만 되돌려져야 한다.

### Edge Cases

- 절대 좌표 노드 이동 중 소스가 외부 변경으로 최신 상태가 아니게 되면 어떻게 반영 충돌을 처리하는가?
- 좌표 값이 음수이거나 매우 큰 수일 때도 최소 변경 원칙을 유지하는가?
- 텍스트 children이 빈 문자열로 변경될 때 의도된 빈 콘텐츠로 저장되는가?
- 텍스트 편집 대상에 비텍스트 children이 혼합되어 있을 때 텍스트 부분만 안전하게 갱신되는가?
- 텍스트/마크다운 오브젝트가 여러 개 있을 때 빠르게 선택 전환하면 비선택 오브젝트가 잘못 편집되지 않는가?
- Markdown 편집 중 표시된 WYSIWYG 결과와 저장 후 렌더 결과가 다를 때 어떻게 처리되는가?
- attach된 스티커/와시테이프의 부모가 삭제 또는 이동된 직후 편집을 시도하면 어떻게 처리되는가?
- 같은 오브젝트를 연속 편집할 때 이전 변경이 반영된 최신 기준으로 누적 적용되는가?
- 드래그 중간 상태가 과도하게 undo 스텝으로 누적되지 않고 편집 완료 이벤트 기준으로 묶이는가?
- 동일 식별자(중복 ID) 충돌이 존재할 때 대상 편집이 차단되고 중복 해결 안내가 제공되는가?

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 절대 좌표 기반 오브젝트 이동 편집을 지원해야 한다.
- **FR-002**: 절대 좌표 오브젝트 이동이 반영될 때 시스템은 해당 오브젝트의 좌표 값만 변경해야 한다.
- **FR-003**: 좌표 편집 시 시스템은 해당 오브젝트의 식별자, 텍스트, 관계, 스타일 등 좌표 외 정보를 변경하지 않아야 한다.
- **FR-004**: 시스템은 Markdown 또는 일반 텍스트 children을 가진 오브젝트에 대해 텍스트 편집을 지원해야 한다.
- **FR-005**: 텍스트 편집 반영 시 시스템은 대상 children 텍스트 내용만 변경해야 한다.
- **FR-006**: 시스템은 다중 줄 텍스트 및 마크다운 문법이 포함된 children 편집을 지원해야 하며, 사용자 미편집 부분은 유지해야 한다.
- **FR-007**: 시스템은 Markdown 편집 시 현재 Markdown 렌더러가 지원하는 문법 범위와 1:1 동일한 WYSIWYG 편집 경험을 제공해야 한다.
- **FR-008**: 시스템은 텍스트/마크다운 편집 가능한 오브젝트가 여러 개일 때 현재 선택된 오브젝트만 편집 대상으로 허용해야 한다.
- **FR-009**: 시스템은 비선택 텍스트/마크다운 오브젝트의 내용이 현재 편집 작업으로 변경되지 않도록 보장해야 한다.
- **FR-010**: 시스템은 attach된 스티커/와시테이프의 위치 편집을 상대 위치 변경으로 반영해야 한다.
- **FR-011**: 상대 위치 편집 시 시스템은 부모-자식 attach 관계를 변경하지 않아야 한다.
- **FR-012**: 시스템은 편집 대상이 유효하지 않거나 최신 기준과 충돌하는 경우, 부분 반영 없이 편집을 거부하고 재동기화(재로드)를 포함한 원인 안내를 제공해야 한다.
- **FR-013**: 시스템은 각 편집 작업마다 목적 범위를 벗어난 부수 변경이 발생하지 않도록 최소 변경 단위를 보장해야 한다.
- **FR-014**: 시스템은 단일 편집 작업 성공 시 코드 재로딩 이후에도 편집 결과가 동일하게 재현되어야 한다.
- **FR-015**: 시스템은 이번 범위에서 신규 오브젝트 생성, 삭제, 재부모화, 스타일 변경을 포함하지 않아야 한다.
- **FR-016**: 시스템은 노드 이동 완료, 텍스트 편집 완료, attach 상대 위치 편집 완료를 각각 독립된 편집 완료 이벤트로 기록해야 한다.
- **FR-017**: 시스템은 각 편집 완료 이벤트마다 undo 1회로 정확히 해당 이벤트 1건만 되돌릴 수 있어야 한다.
- **FR-018**: 시스템은 편집 대상 식별 시 동일 식별자 충돌을 감지하면 편집 반영을 거부하고 중복 해결 안내를 제공해야 한다.
- **FR-019**: 시스템은 redo 1회 실행 시 직전 undo된 편집 완료 이벤트 1건만 복원할 수 있어야 한다.
- **FR-020**: 시스템은 편집 대상 식별자(`id`)를 렌더 결과 전체 범위에서 전역 유일하게 유지해야 하며, 부모-자식 관계 식별자는 부모 식별자 조합으로 전역 유일성을 만족해야 한다.

### Key Entities *(include if feature involves data)*

- **Editable Object**: 캔버스에서 편집 가능한 기존 오브젝트. 절대 좌표 편집 또는 텍스트 편집 대상이 된다.
- **Textual Children Content**: Markdown 또는 일반 텍스트 children으로 표현되는 편집 가능한 콘텐츠 본문.
- **Attached Decoration**: 부모 오브젝트에 attach된 스티커/와시테이프 오브젝트.
- **Relative Offset**: attach된 오브젝트가 부모 기준으로 가지는 상대 위치 값.
- **Edit Change Set**: 한 번의 편집 작업으로 반영되는 최소 코드 변경 묶음.
- **Edit Completion Event**: undo/redo 단위를 구성하는 편집 완료 단위 이벤트.
- **Global Identifier**: 렌더 결과 전체에서 유일해야 하는 오브젝트 식별자.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 절대 좌표 이동 회귀 시나리오의 95% 이상에서 코드 변경이 대상 오브젝트의 좌표 값 변경으로만 검증된다.
- **SC-002**: 텍스트 children 편집 회귀 시나리오의 95% 이상에서 코드 변경이 대상 children 텍스트 변경으로만 검증된다.
- **SC-003**: attach된 스티커/와시테이프 이동 시나리오의 95% 이상에서 코드 변경이 상대 위치 값 변경으로만 검증되고 attach 관계는 유지된다.
- **SC-004**: 사용성 검증에서 편집자 90% 이상이 세 가지 핵심 편집 작업(좌표 이동, 텍스트 수정, attach 상대 이동)을 도움 없이 완료한다.
- **SC-005**: 충돌/유효성 실패 시나리오에서 100% 부분 반영 없이 편집이 거부되고, 실패 원인이 사용자에게 전달된다.
- **SC-006**: Markdown 편집 시나리오의 95% 이상에서 편집 중 WYSIWYG 표시와 저장 후 표시가 동일하게 검증된다.
- **SC-007**: 다중 텍스트/마크다운 오브젝트 편집 시나리오의 99% 이상에서 선택된 오브젝트 외 콘텐츠 변경이 발생하지 않는다.
- **SC-008**: 편집 완료 이벤트 기반 회귀 시나리오의 99% 이상에서 undo 1회가 정확히 직전 이벤트 1건만 되돌리는 것으로 검증된다.
- **SC-009**: 동일 식별자 충돌 시나리오의 100%에서 편집 반영이 차단되고, 중복 해결 안내가 사용자에게 제공된다.
- **SC-010**: 이벤트 기반 히스토리 회귀 시나리오의 99% 이상에서 redo 1회가 직전 undo된 이벤트 1건만 정확히 복원하는 것으로 검증된다.
- **SC-011**: 렌더 결과 전역 유일성 검증 시나리오의 100%에서 식별자 중복이 탐지되고 편집 반영이 차단된다.

## Assumptions

- TSX 코드가 항상 캔버스 상태의 기준 데이터로 유지된다.
- 본 기능은 이미 생성된 오브젝트 편집에만 적용된다.
- 스티커/와시테이프의 attach 관계를 표현하는 현재 데이터 모델은 유효하게 유지되고 있다.

## Dependencies

- 대상 오브젝트를 안정적으로 식별할 수 있는 ID 일관성이 보장되어야 한다.
- 편집 반영 전후 변경 내역을 비교할 수 있는 검증 흐름이 제공되어야 한다.
- 편집 실패 사유를 사용자에게 전달할 수 있는 공통 오류 노출 방식이 존재해야 한다.

## Out of Scope

- 신규 오브젝트 생성 및 삭제
- 오브젝트 재부모화 또는 관계 구조 변경
- 스타일, 타입, 기타 속성의 일괄 편집
- 자동 레이아웃/정렬 규칙 변경
