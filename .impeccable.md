## Design Context

### Users
Primary users are first-time developers encountering Magam through the browser-first web demo. They arrive with little prior context and want to understand the product quickly by browsing examples, reading code, and seeing immediate visual output. Their core job is to decide, within a short session, whether Magam feels powerful, understandable, and fast enough to explore further.

### Brand Personality
The interface should feel clean, clear, and simple. It should create confidence by removing noise, spark curiosity through immediate visual feedback, and communicate speed through direct, low-friction interactions. The tone should feel calm and precise rather than playful, flashy, or overly "designy."

### Aesthetic Direction
Use a restrained, browser-native-feeling product UI with strong information hierarchy and minimal ornament. Favor crisp surfaces, readable typography, obvious separation between navigation, code, and preview, and spacing that makes the interface feel breathable without becoming sparse. Avoid generic SaaS dashboard polish, glowing dark-mode theatrics, and decorative treatment that competes with the example content itself.

### Design Principles
1. Make the first five seconds legible: the current example, the code, and the preview should all be understandable at a glance.
2. Use spacing to create confidence: tight where information belongs together, generous where a mental boundary matters.
3. Keep the shell quiet so the rendered example stays primary.
4. Show speed through responsiveness and direct feedback, not through flashy effects.
5. Prefer plain, explicit UI language and visual affordances over cleverness or abstraction.
